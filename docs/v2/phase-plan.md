# KnowLink 第二版阶段计划与负责人分工

日期：2026-05-13

## 1. 分工口径

本文只用于第二版功能规划、任务拆分和负责人说明，不展开具体实施步骤。

第二版按以下责任口径推进：

- 朱春雯：全权负责前端，包括 Flutter 页面、Android 运行、页面设计优化、前端交互和前端联调。
- 曹乐：V2 后端技术主责与 AI 主责，负责所有困难后端、AI、解析、B站导入核心、知识图谱、流式输出、主观题判卷和推荐策略。
- 杨彩艺：后端辅助与用户测试调研主责，负责边界清楚、低风险、可独立完成的后端辅助任务，包括基础 CRUD、简单 DTO、状态查询、接口文档、测试数据整理、简单接口联调；第三阶段总负责真实用户测试反馈调研。

第二版不做多用户，只做单用户下的多课程管理。

本文件是第二版阶段路线图和 V2 任务责任口径。`docs/v1/team-division.md` 已标注为 V1 分工文档；V2 中若旧 owner 口径与本文冲突，以本文为准，不得把本文中明确归曹乐主责的困难后端任务默认分配给杨彩艺。

第二版执行时，困难后端任务不默认分配给杨彩艺；若任务涉及复杂状态机、任务队列、下载合并、AI 产物、复杂解析、知识图谱、流式事件或主观题判卷逻辑，默认由曹乐负责。杨彩艺只接已经由曹乐拆清楚输入、输出、状态和验收证据的辅助后端任务。

## 3. 阶段一：真实可用闭环

目标：把第一版从“核心流程可演示”推进到“真实资料、真实移动端、真实课程导入可用”。

### 阶段一功能范围

- B站真实登录、下载、导入。
- B站范围包含单视频、多 P、合集、番剧。
- 不做收藏夹、热门列表、大规模批量下载。
- 多课程基础管理。
- 课程库设计与补全。
- 智能课程推荐基础版。
- Android 环境运行。
- 页面设计优化。
- 复杂版面增强的最低可用门槛。

## 4. 阶段二：AI 与图谱重能力

目标：补齐第二版的核心智能能力，让课程学习不只是“资料展示”，而是能围绕知识点、图谱、实时反馈和主观题评价运转。

### 阶段二功能范围

- 复杂知识图谱。
- 实时流式输出。
- 主观题自动判卷。
- 多课程管理增强。
- 智能课程推荐增强版。
