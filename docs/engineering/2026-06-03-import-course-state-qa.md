# 2026-06-03 导入页课程状态 QA 记录

## 版本与边界

- 记录类型：真实浏览器用户测试问题记录。
- 版本判断：涉及课程推荐确认、课程详情和自主导入页，属于 V2 多课程管理与推荐入课联调路径。
- Owner 边界：本文件只记录现象和证据，不改接口 contract、schema、业务实现或 owner 分工。疑似问题落在 Flutter 前端导入页状态展示；后端课程创建和查询已在本次测试中返回正确数据。

## 测试环境

- 日期：2026-06-03
- 后端启动方式：`docker compose up --build -d api worker`
- 后端地址：`http://127.0.0.1:8000`
- 前端地址：`http://127.0.0.1:52853`
- 浏览器：Chrome，通过真实页面点击完成测试

## 复现步骤

1. 打开前端首页 `http://127.0.0.1:52853`。
2. 进入底部导航「推荐」。
3. 点击「获取推荐」。
4. 在推荐课程中选择「高等数学期末冲刺」，点击「确认入课并导入资料」。
5. 页面显示课程创建成功，生成课程 ID `48`。
6. 点击「前往自主导入页」，进入 `#/import?courseId=48`。

## 实际结果

- URL 和右侧上传面板都指向课程 `48`。
- 右侧上传面板显示「当前课程：48」。
- 后端 `GET /api/v1/courses/48` 返回的课程标题为「高等数学期末冲刺」。
- 首页「最近学习」第一项显示「高等数学期末冲刺」。
- 课程详情页 `#/courses/48` 正确显示「高等数学期末冲刺」和 `recommendation` 来源。
- 但导入页左侧「课程信息」表单仍显示默认课程信息：
  - 课程名称：`KnowLink 固定联调课`
  - 学习目标：`期末复习`
  - 讲义风格偏好：`平衡讲解`

## 期望结果

从推荐确认后的课程进入自主导入页时，导入页左侧课程信息应与当前 `courseId=48` 保持一致，至少不应展示另一个默认课程的名称和目标。可接受的产品表现包括：

- 按当前课程回填课程名称、学习目标和偏好；
- 或将左侧表单明确标识为「创建新课程」，并避免与右侧「当前课程」产生冲突；
- 或在已绑定 `courseId` 的导入路径中隐藏创建课程表单，仅保留当前课程摘要和上传入口。

## 影响判断

- 严重级别：P2。
- 用户影响：用户从推荐入课后会看到同一页面同时出现课程 `48` 和默认固定联调课信息，容易误以为资料会上传到错误课程。
- 数据影响：本次测试未发现后端数据写错。课程 `48` 在首页、详情页和 API 查询中均保持正确。

## 相关验证

- Chrome 控制台：测试路径中未捕获 error 级日志。
- API 健康检查：`GET /health` 返回 `200 {"status":"ok"}`。
- Docker 服务：`api`、`worker`、`postgres`、`redis`、`minio` 均处于运行状态。
- 后端日志：推荐获取、确认入课、课程详情、课程资源、QA、解析进度等请求未出现 5xx。
- 空课程进入讲义页时出现业务态响应：`GET /api/v1/courses/48/handouts/latest` 返回 404，`POST /api/v1/courses/48/handouts/generate` 返回 409；页面未崩溃。
